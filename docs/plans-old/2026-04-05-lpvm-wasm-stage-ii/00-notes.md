# M2 Stage II: lpvm-wasm Browser Runtime + Direct Builtin Linking

## Scope

1. Replace the current "load builtins `.wasm` from filesystem" approach with
   direct linking of `lps-builtins` (Rust dependency).
2. Add a browser runtime backend (`js_sys::WebAssembly`) implementing the LPVM
   traits (`LpvmEngine`, `LpvmModule`, `LpvmInstance`) for `wasm32` targets.
3. Update `web-demo` to use `lpvm-wasm` instead of `lps-wasm`, validating the
   browser runtime end-to-end (full runtime in Rust).

## Decisions

### Direct Linking (not embedded bytes)

`lpvm-wasm` depends on `lps-builtins` directly. No separate `.wasm` file needed.

**Host (wasmtime):** Use `wasmtime::Func::new` with dynamic `FuncType` to
provide each builtin as a host function. Match shader imports by name against
`lps-builtin-ids` registry. Builtins run at native speed.

**Browser (wasm32):** `#[no_mangle] pub extern "C"` builtin functions from
`lps-builtins` appear as WASM exports in the final cdylib binary. The browser
runtime receives its own exports via an init call:

```js
wasm.lpvm_init_exports(instance.exports);
```

Rust stores the exports, extracts `WebAssembly.Function` objects by name, and
passes them as imports when instantiating shader modules. Browsers can optimize
these cross-module calls to near-direct speed.

### No runtime feature flag

The runtime is always included. `lpvm-wasm` is a JIT — compile and run in the
same process. Target selects the backend:
- `!wasm32` → wasmtime
- `wasm32` → js_sys/wasm-bindgen

### Parallel runtime directories

`rt_wasmtime/` (host) and `rt_browser/` (wasm32). Cfg-gated in `lib.rs`.
Each is self-contained. Shared constants/type mapping lives in `module.rs`.

### Full runtime in Rust for web-demo

web-demo exposes compile + instantiate + render_frame via wasm-bindgen.
JS only handles UI, canvas blitting, and requestAnimationFrame. Validates
the entire LPVM trait chain in-browser.

### lps-builtins-wasm kept in legacy/

Already moved to `legacy/`. `lps-filetests` still uses it until M5.

### DCE prevention in lps-builtins

Move `ensure_builtins_referenced()` into `lps-builtins` itself (generated by
`lps-builtins-gen-app`). Any consumer calls it to prevent DCE. `lps-builtins-wasm`
drops its own copy and uses the upstream one.
