# Q32 — Q16.16 Fixed-Point Arithmetic

This document is the single source of truth for Q32 numeric semantics in
LightPlayer. All implementations (reference `Q32` struct, JIT builtins,
Cranelift emitter, WASM emitter, LPIR interpreter Q32 mode) must conform.

## 1. Encoding

Q32 is a **signed Q16.16** fixed-point format stored in an `i32`.

- **16 integer bits** (signed): range −32768 to 32767
- **16 fractional bits**: resolution 1/65536 ≈ 0.0000153

```
  bit 31 (sign)
  ┌─┐
  │S│ integer bits (15)  │ fractional bits (16)  │
  └─┴────────────────────┴───────────────────────┘
  31                    16 15                     0
```

**Raw representation:** the stored `i32` equals `floor(value × 65536)` for
non-negative values; two's complement for negative.

**Representable range:** `[−32768.0, 32767.999984741]` (i.e. `i32::MIN` to
`0x7FFF_FFFF` as raw values).

**Scale constant:** `ONE = 1 << 16 = 65536`.

## 2. Conversions

| Conversion         | Rule                                                                                                         |
|--------------------|--------------------------------------------------------------------------------------------------------------|
| `from_f32(f)`      | `(f * 65536.0) as i32` — truncates toward zero                                                               |
| `from_i32(i)`      | `i << 16` — caller must ensure `i` fits in 16 signed bits                                                    |
| `from_fixed(raw)`  | Identity — wraps raw `i32` as Q32                                                                            |
| `to_f32()`         | `raw as f32 / 65536.0`                                                                                       |
| `to_i32()`         | Floor toward −∞: arithmetic right shift `raw >> 16` (matches `floor_vec` / GLSL `floor` on the integer grid) |
| `to_u8_clamped()`  | `to_i32().clamp(0, 255) as u8`                                                                               |
| `to_u16_clamped()` | `((raw as i64 * 65535) / 65536).clamp(0, 65535) as u16`                                                      |

`from_f32` does **not** saturate — values outside representable range wrap via
the `as i32` cast. This matches Rust's `as` semantics. The Cranelift emitter's
`q32_encode` function does saturate (for constant encoding), but the struct
conversion does not.

## 3. Arithmetic

All Q32 arithmetic operators **saturate** to `[i32::MIN, 0x7FFF_FFFF]`. This
matches the JIT `extern "C"` builtins and prevents wrapping overflow, which
would produce catastrophic visual artifacts in shaders.

### Addition / Subtraction

Widen both operands to `i64`, perform the operation, clamp result to
`[i32::MIN, 0x7FFF_FFFF]`, truncate to `i32`.

### Multiplication

Widen to `i64`, multiply, arithmetic right-shift by 16, clamp to
`[i32::MIN, 0x7FFF_FFFF]`, truncate to `i32`.

### Division

Widen dividend to `i64`, shift left by 16, divide by (widened) divisor,
clamp result to `[i32::MIN, 0x7FFF_FFFF]`.

**Division by zero:**

| Case           | Result                            |
|----------------|-----------------------------------|
| `0 / 0`        | `Q32(0)`                          |
| `positive / 0` | `Q32(0x7FFF_FFFF)` (max positive) |
| `negative / 0` | `Q32(i32::MIN)` (max negative)    |

Rationale: sign-preserving saturation keeps downstream math (e.g.
`clamp(x/0, 0, 1)`) visually plausible. `0/0` has no meaningful sign →
zero.

### Remainder

`x % y` operates on raw `i32` values.

**Remainder by zero:** returns `Q32(0)`. GLSL `mod(x, 0.0)` is undefined;
zero is safe.

### Negation

`-x` is `Q32(-raw)`. No saturation needed (negating `i32::MIN` wraps to
`i32::MIN` in two's complement, which is the only edge case — it represents
−32768.0, and negating it would need +32768.0 which is outside range; the
wrap to `i32::MIN` is the closest representable value).

### Absolute value

`abs(x)` is `Q32(raw.abs())`. Same edge case as negation: `abs(i32::MIN)` →
`i32::MIN` (Rust's `i32::abs()` panics in debug; the implementation should
use `wrapping_abs` or handle the edge explicitly).

## 4. Named Constants

| Name   | Intended value  | Raw `i32` | Verified                |
|--------|-----------------|-----------|-------------------------|
| `ZERO` | 0.0             | `0`       | `0.0 × 65536 = 0`       |
| `HALF` | 0.5             | `32768`   | `0.5 × 65536 = 32768`   |
| `ONE`  | 1.0             | `65536`   | `1.0 × 65536 = 65536`   |
| `PI`   | π ≈ 3.14159265  | `205887`  | `π × 65536 ≈ 205887.4`  |
| `TAU`  | 2π ≈ 6.28318531 | `411774`  | `2π × 65536 ≈ 411774.7` |
| `E`    | e ≈ 2.71828183  | `178145`  | `e × 65536 ≈ 178145.1`  |
| `PHI`  | φ ≈ 1.61803399  | `106039`  | `φ × 65536 ≈ 106039.0`  |

## 5. GLSL Builtins on Q32

Builtins implemented in the reference `fns/` module and as `extern "C"` JIT
builtins. All operate component-wise on vectors.

| Builtin            | Q32 behavior                                                                                             |
|--------------------|----------------------------------------------------------------------------------------------------------|
| `sin(x)`           | Polynomial approximation via `__lps_sin_q32`                                                             |
| `cos(x)`           | Polynomial approximation via `__lps_cos_q32`                                                             |
| `floor(x)`         | Mask fractional bits; adjust for negative                                                                |
| `fract(x)`         | `x & (ONE - 1)` (extract fractional bits)                                                                |
| `mod(x, y)`        | GLSL `mod`: `x - y * floor(x/y)` via `__lps_mod_q32`                                                     |
| `min(a, b)`        | Integer signed comparison on raw values                                                                  |
| `max(a, b)`        | Integer signed comparison on raw values                                                                  |
| `clamp(x, lo, hi)` | `max(lo, min(hi, x))`                                                                                    |
| `mix(a, b, t)`     | `a + t * (b - a)` using Q32 arithmetic                                                                   |
| `step(edge, x)`    | `edge <= x ? ONE : ZERO` (integer compare)                                                               |
| `sqrt(x)`          | Integer square root via `__lp_lpir_fsqrt_q32`: `x <= 0 → 0`                                              |
| `abs(x)`           | See §3                                                                                                   |
| `sign(x)`          | `x > 0 → ONE`, `x < 0 → -ONE`, `x == 0 → ZERO`                                                           |
| `roundEven(x)`     | Round to nearest integer, halfway to even, via `__lp_lpir_fnearest_q32`                                  |
| `round(x)`         | Round to nearest integer, **halfway cases away from zero** (matches GLSL `round`), via `__lps_round_q32` |

### Builtins not yet implemented

Builtins that exist in GLSL but are not yet in the Q32 reference:
`smoothstep`, `exp`, `exp2`, `log`, `log2`, `pow`, `inversesqrt`, `atan`,
`asin`, `acos`, `sinh`, `cosh`, `tanh`, `ceil`, `trunc`.

Some of these have Cranelift emitter implementations (`emit_fceil`,
`emit_ftrunc` in `q32.rs`) but no reference `Q32` struct method.

## 6. Relational Builtins

| Builtin     | Q32 behavior          | Rationale               |
|-------------|-----------------------|-------------------------|
| `isnan(x)`  | Always `false`        | Q32 has no NaN encoding |
| `isinf(x)`  | Always `false`        | Q32 has no Inf encoding |
| `all(bvec)` | `iand` chain on lanes | Same as IEEE path       |
| `any(bvec)` | `ior` chain on lanes  | Same as IEEE path       |
| `not(bvec)` | Per-component `== 0`  | Same as IEEE path       |

The `0x7FFF_FFFF` and `i32::MIN` values produced by division-by-zero
saturation are **not** exposed as infinity through `isinf`. They are
implementation-internal saturation artifacts.

## 7. `@unsupported` Policy

Filetests that require real IEEE NaN or Inf behavior use the annotation
`@unsupported(float_mode=q32, reason="...")`. These tests are skipped when
running under Q32 mode. This is preferable to making Q32 pretend to be IEEE.

Examples of `@unsupported` cases:

- `isinf(1.0/0.0) == true` (IEEE: true, Q32: false)
- NaN propagation through arithmetic
- Inf propagation through builtins (sin, log, exp of infinity)

## 8. Comparisons

Q32 comparisons are **signed integer comparisons** on the raw `i32` payload.
This produces correct results because Q16.16 is monotonic: if `a < b` as real
numbers, then `a.0 < b.0` as signed integers.

| GLSL op | LPIR op | Q32 lowering                    |
|---------|---------|---------------------------------|
| `<`     | `Flt`   | `icmp SignedLessThan`           |
| `<=`    | `Fle`   | `icmp SignedLessThanOrEqual`    |
| `>`     | `Fgt`   | `icmp SignedGreaterThan`        |
| `>=`    | `Fge`   | `icmp SignedGreaterThanOrEqual` |
| `==`    | `Feq`   | `icmp Equal`                    |
| `!=`    | `Fne`   | `icmp NotEqual`                 |

## 9. Backend Conformance

All backends must produce results matching the semantics in this document:

- **Cranelift JIT** (`lpvm-cranelift`): Q32 mode emits integer instructions
  with saturating arithmetic builtins. `Q32Options` controls per-shader mode
  selection (saturating vs wrapping add/sub/mul, saturating vs reciprocal div).
  Default: all saturating.
- **WASM** (`lps-wasm`): must conform to the same semantics. May use
  `i64` intermediate arithmetic internally. Implementation details are in
  `lps-wasm` crate docs.
- **LPIR interpreter** (`lpir::interp`): when running in Q32 mode, must match.
  Currently runs as f32; Q32 interpreter mode is future work.

## 10. Reference Implementation

The canonical Rust reference is:

- **`Q32` struct**: `lp-shader/lps-builtins/src/glsl/q32/types/q32.rs`
- **Builtin functions**: `lp-shader/lps-builtins/src/glsl/q32/fns/`
- **JIT builtins**:
  `lp-shader/lps-builtins/src/builtins/lpir/{fadd,fsub,fmul,fdiv,fsqrt,fnearest}_q32.rs`
- **Cranelift emitter helpers**: `lp-shader/legacy/lpvm-cranelift/src/q32.rs`

The `Q32` struct is the primary reference for scalar behavior. JIT builtins
are `extern "C"` wrappers with identical semantics (they operate on raw `i32`
for ABI reasons). The Cranelift emitter helpers implement inline Q32
operations that don't warrant a function call (neg, abs, min, max, floor,
ceil, trunc, comparisons, integer conversions).

Tests in the `Q32` struct module and in the JIT builtin modules lock edge-case
behavior. Filetests in `lps-filetests` provide end-to-end validation.
