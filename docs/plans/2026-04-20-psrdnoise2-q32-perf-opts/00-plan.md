# `psrdnoise2_q32` perf opts — multi-phase plan

**Goal:** Drive `__lp_lpfn_psrdnoise2_q32` from 35.3% self toward roughly half that, by applying the seven optimizations from the analysis in distinct phases. Each phase: implement, add/update snapshot regression tests, run `just check && just test`, capture a `lp-cli profile examples/perf/fastmath` snapshot, and commit. All implementation phases run on Composer 2 sub-agents.

**Profile target for every snapshot:** `examples/perf/fastmath` (frozen baseline copy of `examples/basic` per [examples/perf/fastmath/README.md](../../../examples/perf/fastmath/README.md)). Note tag for each phase is given below.

**Working directory for everything:** workspace root.

### Important: take your own baseline

Concurrent perf work is happening on other branches / commits. Any `p0-baseline` profile directory already present under `profiles/` when this plan starts executing is **stale** and must be ignored. Phase 0 captures a fresh `p0-baseline` profile against `HEAD` at the moment the phase runs, and every subsequent phase must compare against **that** captured profile, not against any pre-existing one. Same applies to the Phase 6 final comparison: it must compare `p0-baseline` and `p6-final` directories that were both produced during this plan's execution.

Snapshot file format (plain text, one line per probe, sorted, raw `i32`):

```
# Auto-generated by lpfn_q32_snapshots tests. Update by setting LP_UPDATE_SNAPSHOTS=1.
psrdnoise2_q32(x=98304,y=151552,px=0,py=0,a=0,seed=0) = noise=18372 grad=(12345,-67890)
psrdnoise2_q32(x=-65536,y=131072,px=655360,py=655360,a=32768,seed=0) = noise=-8421 grad=(...,...)
...
```

Probe grids are deterministic and small (~32 probes per fn), chosen to cover: positive/negative inputs, integer/fractional inputs, period 0 vs nonzero, alpha 0 vs nonzero, edge cases near 0. Each test renders to a `String` and either asserts equality with `include_str!`-loaded snapshot, or, if `LP_UPDATE_SNAPSHOTS=1` is set, overwrites the file and fails with a "snapshot updated" message (so CI never silently accepts).

Helper sketch:

```rust
pub fn assert_snapshot(name: &str, actual: &str) {
    let path = format!("tests/snapshots/lpfn_q32/{name}.snap.txt");
    if std::env::var("LP_UPDATE_SNAPSHOTS").is_ok() {
        std::fs::write(&path, actual).unwrap();
        panic!("wrote {path}; rerun without LP_UPDATE_SNAPSHOTS");
    }
    let expected = std::fs::read_to_string(&path).unwrap_or_default();
    assert_eq!(expected, actual, "snapshot mismatch for {name}");
}
```

## Phase 1 — Integer hash chain in `psrdnoise2_q32` [sub-agent: yes]

The biggest single win.

File: [lp-shader/lps-builtins/src/builtins/lpfn/generative/psrdnoise/psrdnoise2_q32.rs](../../../lp-shader/lps-builtins/src/builtins/lpfn/generative/psrdnoise/psrdnoise2_q32.rs).

Replace the 9 `__lps_mod_q32` calls + ~12 `Q32 * Q32` multiplies in the three hash blocks (lines ~166–210) with `i32`-only arithmetic, since hash values are bounded in `[0, 288]` and intermediates fit comfortably in `i32`:

```rust
#[inline(always)]
fn hash_corner(iu: i32, iv: i32) -> i32 {
    let h = iu.rem_euclid(289);                                // [0, 288]
    let h = ((h * 51 + 2) * h + iv).rem_euclid(289);           // intermediate <= ~4.2M
    ((h * 34 + 10) * h).rem_euclid(289)                        // intermediate <= ~2.8M
}

let hash_x = hash_corner(iu_x, iv_x);
let hash_y = hash_corner(iu_y, iv_y);
let hash_z = hash_corner(iu_z, iv_z);

// psi_raw = hash * 4904 + alpha_raw  (hash * HASH_MULT_0_07482.0 fits in i32)
let psi_x = Q32::from_fixed(hash_x.wrapping_mul(HASH_MULT_0_07482.0).wrapping_add(alpha.0));
let psi_y = Q32::from_fixed(hash_y.wrapping_mul(HASH_MULT_0_07482.0).wrapping_add(alpha.0));
let psi_z = Q32::from_fixed(hash_z.wrapping_mul(HASH_MULT_0_07482.0).wrapping_add(alpha.0));
```

Note: the existing Q32 hash silently saturates on `(hash * 51 + 2) * hash` (intermediate raw ~ 2.77e11), so snapshots WILL change in this phase — the new values are actually closer to the GLSL reference. This is expected and correct.

### Test + snapshot updates

1. `cargo test -p lps-builtins` — existing unit tests in `psrdnoise2_q32.rs` should still pass (range checks + determinism).
2. Snapshot file `psrdnoise2_q32.snap.txt` will fail; regenerate with `LP_UPDATE_SNAPSHOTS=1 cargo test -p lps-builtins --test lpfn_q32_snapshots`.
3. Re-run cargo test for clean snapshot pass.
4. Run filetests for psrdnoise: `scripts/filetests.sh --target jit.q32` then `--target rv32c.q32` (the existing [lp-shader/lps-filetests/filetests/lpfn/lp_psrdnoise.glsl](../../../lp-shader/lps-filetests/filetests/lpfn/lp_psrdnoise.glsl) checks tolerance-based behaviour, should still pass).
5. Profile: `cargo run -p lp-cli --release -- profile examples/perf/fastmath --note p1-int-hash`.
6. Commit message body must call out that snapshot deltas are expected and explain why (existing Q32 hash saturated; new path matches reference).

**Validate:** `just check`, `cargo test -p lps-builtins`, `scripts/filetests.sh --target jit.q32`.

## Phase 2 — Q32 micro-ops in psrdnoise2 [sub-agent: yes]

Bundle the trivially safe micro-optimizations. Snapshots **must not** change here; if they do, that's a bug.

Touched files:

- [lp-shader/lps-q32/src/q32.rs](../../../lp-shader/lps-q32/src/q32.rs) — add `pub const fn half(self) -> Q32 { Q32(self.0 >> 1) }` (mathematically identical to `* HALF`).
- [lp-shader/lps-builtins/src/builtins/lpfn/generative/psrdnoise/psrdnoise2_q32.rs](../../../lp-shader/lps-builtins/src/builtins/lpfn/generative/psrdnoise/psrdnoise2_q32.rs):
  - Replace every `* HALF` (lines 90, 118, 120, 122, 152–158) with `.half()`.
  - Branchless `cmp = step(f0.y, f0.x)` using sign of `(f0.x.0 - f0.y.0)`:
    ```rust
    let cmp_raw = (((f0.x.0 - f0.y.0) >> 31).wrapping_add(1)) << 16; // 0 or 0x10000
    let cmp = Q32(cmp_raw);
    ```
  - Lift the periodic guard: keep the runtime `period.x > 0 || period.y > 0` branch, but factor the no-period path into an `#[inline(always)] fn psrdnoise2_noperiod(...)` so the compiler/JIT call site can constant-fold when callers pass `vec2(0.0)` (e.g. via `extern "C" __lp_lpfn_psrdnoise2_q32` checking the period args once at the top and dispatching).

2. Snapshot tests must pass **without** regeneration. If a delta appears, stop and report — bug.
3. Profile: `--note p2-microops`. Commit.

## Phase 4 — Cos/sin LUT for gradient angles [sub-agent: yes]

New file: `lp-shader/lps-builtins/src/builtins/lpfn/generative/psrdnoise/grad_lut_q32.rs`.

Build a `const` table of `(cos_q32, sin_q32)` indexed by `hash in [0, 288]`, computing each entry at compile time from `hash * 0.07482`. Replace the per-corner sincos with one LUT lookup + alpha rotation identity using a single `(sin α, cos α)` pair computed once at function entry:

```rust
let (sin_a, cos_a) = __lps_sincos_q32(alpha.to_fixed());
let (c0, s0) = GRAD_LUT[hash_x as usize];
let gx_x = ((c0 as i64 * cos_a as i64 - s0 as i64 * sin_a as i64) >> 16) as i32;
let gy_x = ((s0 as i64 * cos_a as i64 + c0 as i64 * sin_a as i64) >> 16) as i32;
// repeat for hash_y, hash_z
```

Per-call trig drops from 6 Taylor series -> 1 sincos call + 6 lookups + 12 muls + 6 add/subs.

LUT size: 289 × 2 × 4 bytes = 2312 B in `rodata`. Document in module-level comment.

### Test + snapshot

1. Cargo test (psrdnoise2 unit tests still range-pass).
2. Snapshots: psrdnoise2 will shift; regenerate. The numerical delta should be small (~LUT precision); spot-check a few values vs. the float reference in the snapshot test (optional sanity assert with 0.05 absolute tolerance vs. the f32 `psrdnoise2_f32` for matching probe points).
3. Filetests `lp_psrdnoise.glsl` use 0.001/0.0001 tolerances on differences — verify they still pass; if any tolerance-based assertion newly fails, escalate to user (don't widen tolerances).
4. Profile: `--note p4-trig-lut`. Commit.

**Validate:** `just check`, `cargo test -p lps-builtins`, `scripts/filetests.sh --target jit.q32 --target rv32c.q32`.

- [lp-shader/lps-q32/src/q32.rs](../../../lp-shader/lps-q32/src/q32.rs): add `pub fn mul_wrapping(self, rhs: Q32) -> Q32` returning `Q32(((self.0 as i64 * rhs.0 as i64) >> 16) as i32)`. Optional `add_wrapping`/`sub_wrapping` if needed.
