# Milestone 0: Foundation Refactor — Profile Session

## Goal

Introduce the unified profiling infrastructure (`Collector` trait,
`ProfileSession`, unified trace dir layout) and port the existing
alloc-trace into it. Ship `lp-cli profile --collect alloc` at full
parity with the existing `mem-profile` command. Remove `mem-profile`
and `heap-summary` commands. **No new functionality** — pure
restructuring laying the foundation for m1 onward.

## Suggested Plan Name

`profile-m0-foundation`

## Scope

### In scope

- **`Collector` trait** in new module `lp-riscv-emu/src/profile/mod.rs`.
  Designed once, never revised: m1's `CpuCollector` and `EventsCollector`
  plug into the same trait without changes.

  ```rust
  pub trait Collector: Send {
      fn name(&self) -> &'static str;
      fn on_syscall(
          &mut self,
          _emu: &mut Riscv32Emulator,
          _id: u32,
          _args: &[u32],
      ) -> SyscallAction { SyscallAction::Pass }
      fn on_instruction(&mut self, _pc: u32, _kind: InstClass, _cycles: u32) {}
      fn on_perf_event(&mut self, _evt: &PerfEvent) {}
      fn finish(&mut self, ctx: &FinishCtx) -> std::io::Result<()>;
      fn report_section(&self, w: &mut dyn std::fmt::Write) -> std::fmt::Result;
  }
  ```

  m0 only exercises `on_syscall`, `finish`, `report_section`. The
  `on_instruction` and `on_perf_event` methods exist as no-op defaults
  so m1 collectors can fill them in without trait changes.

- **`ProfileSession`** struct in `profile/mod.rs`. Owns the enabled
  collectors plus shared trace dir state (path, `meta.json` writer,
  combined `report.txt` writer). Replaces direct `Option<AllocTracer>`
  ownership on `Riscv32Emulator`. Single dispatch point for syscalls,
  instruction events, and finish.

- **Port `AllocTracer` → `AllocCollector`** in new file
  `lp-riscv-emu/src/profile/alloc.rs`. **Wire format preserved
  byte-for-byte** — `heap-trace.jsonl` and `meta.json` unchanged.
  Internal struct gets renamed; data shape, file names, JSON field
  names all stable. Old `lp-riscv-emu/src/alloc_trace.rs` deleted.

- **Unified trace dir layout** finalized. Format:
  `traces/<timestamp>--<workload>[--<note>]/`. Matches existing
  convention from `mem-profile`. m1 will add files into this layout;
  it will not change the layout itself.

  ```
  traces/2026-04-19T15-30-22--examples-basic/
    meta.json                # session metadata + symbols (was per-tracer; now shared)
    heap-trace.jsonl         # written when alloc collector ran
    report.txt               # combined; alloc contributes "Heap summary" section
  ```

- **Combined `report.txt`** generated by `ProfileSession::finish()`.
  Iterates enabled collectors calling `report_section()`; concatenates
  with section dividers. Printed to stdout AND written to disk.
  Replaces what `heap-summary` did as a separate command. The text
  content of the alloc section matches `heap-summary`'s current output
  verbatim (port the formatting code).

- **New `lp-cli profile` command** — single top-level command. m0
  surface:

  ```
  lp-cli profile [DIR=examples/basic]
                 [--collect alloc]      # only `alloc` valid in m0; other values rejected
                 [--frames N=10]        # temporary: m1 replaces with --mode
                 [--note STR]

  lp-cli profile diff <a> <b>           # stub: prints "implemented in m2"; reserved
  ```

  `--collect` accepts a comma-separated list; m0 only validates `alloc`.
  m1 expands to `cpu`, `events`, `cpu,alloc`, etc. The arg parser is
  forward-compatible so m1 doesn't redo it. `profile diff` exists as a
  stub subcommand so the command tree shape is stable from m0; m2
  wires up the implementation.

- **Remove old commands.** `lp-cli mem-profile` and `lp-cli
  heap-summary` deleted, including their `commands/mem_profile/` and
  `commands/heap_summary/` modules and registrations in
  `lp-cli/src/main.rs`.

- **Cargo feature rename in `fw-emu`.** Existing `alloc-trace` feature
  renamed to `profile`. Same guest-side syscall behavior; just the
  feature name changes so m1's perf-event syscall additions slot under
  the same gate.

- **Tests.** Existing alloc-trace tests in `lp-riscv-emu` rewired to
  construct an `AllocCollector` inside a `ProfileSession`. Wire-format
  assertions kept (verifies preservation). New CLI smoke test exercises
  `lp-cli profile --collect alloc` against `examples/basic` and
  asserts `traces/<sess>/heap-trace.jsonl` is created and parseable.

### Out of scope

- CPU collector, events collector, perf-event syscall (all m1).
- `--mode` flag and `ProfileMode` enum (m1).
- Functional `profile diff` command (m2 — stub only here).
- `--diff [PATH]` flag on `profile` (m2).
- Hardware sink, console parser, correlation (m3).
- JIT symbol overlay (m4).
- Documentation under `docs/design/native/fw-profile/` (m5).
- Any change to `AllocCollector`'s data model or output format.
  *Pure rename + retraiting.*

## Key Decisions

- **Trait shape committed in m0, not m1.** The cost of getting
  `Collector` slightly wrong in m0 is one minor refactor before m1;
  the cost of getting it visibly evolving across multiple milestones
  is a confused public API. Better to design once with m1's needs
  already in mind.

- **Wire format preservation is non-negotiable.** Even though there are
  no production users of `mem-profile`, agents and developers may have
  prior trace dirs they want to spot-check after the rename.
  `heap-trace.jsonl` schema must round-trip identically.

- **Combined `report.txt` from m0**, not "alloc section only." The
  combined report is the unified-tool deliverable that m1+ collectors
  append to. Get the section-concatenation pattern right with one
  collector before adding two.

- **`profile diff` is a stub from m0.** Reserves the sub-subcommand,
  exits with a clear "not yet implemented (m2)" message. Avoids
  command-tree churn between milestones.

- **Cargo feature name `profile`** (not `alloc-trace` or
  `profile-alloc`). One feature gates all profile-related guest-side
  cooperation. m1 adds perf-event syscall handling under the same
  feature.

- **No `lp-engine` changes in m0.** All work is in `lp-riscv-emu`,
  `lp-cli`, `fw-emu`. `lp-engine` is touched first in m1 (perf-event
  emission).

## Deliverables

- New: `lp-riscv-emu/src/profile/mod.rs` — `Collector` trait,
  `ProfileSession`, `FinishCtx`, `SyscallAction`.
- New: `lp-riscv-emu/src/profile/alloc.rs` — `AllocCollector`
  implementing `Collector`. Contains the formatting code that was
  previously in `lp-cli/src/commands/heap_summary/`.
- Deleted: `lp-riscv-emu/src/alloc_trace.rs`.
- Updated: `lp-riscv-emu/src/lib.rs` — exports
  `profile::{Collector, ProfileSession, AllocCollector}`; `alloc_trace`
  re-exports removed.
- Updated: `lp-riscv-emu/src/emu/emulator/state.rs` — replaces
  `alloc_tracer: Option<AllocTracer>` field with `profile_session:
  Option<ProfileSession>`. Replaces `with_alloc_trace` /
  `finish_alloc_trace` with `with_profile_session` /
  `finish_profile_session`.
- Updated: `lp-riscv-emu/src/emu/emulator/run_loops.rs` — replaces
  direct `alloc_tracer` calls with `profile_session.dispatch_*`
  helpers.
- New: `lp-cli/src/commands/profile/` directory with `mod.rs`,
  `args.rs`, `handler.rs`, `diff_stub.rs`.
- Deleted: `lp-cli/src/commands/mem_profile/`,
  `lp-cli/src/commands/heap_summary/`.
- Updated: `lp-cli/src/main.rs` — registers `profile`, removes
  `mem-profile` and `heap-summary`.
- Updated: `lp-fw/fw-emu/Cargo.toml` — feature `alloc-trace` renamed
  to `profile`.
- Updated: any test fixtures, CI scripts, dev docs that referenced the
  old command names. (Per the user note that there are no production
  users; internal references still need updating.)

## Dependencies

None — first milestone of the roadmap.

## Validation

```bash
# Existing alloc-trace tests still pass through new infrastructure
cargo test -p lp-riscv-emu

# New CLI command exists and works
cargo run -p lp-cli -- profile examples/basic --collect alloc --frames 10
# Verify: traces/<timestamp>--examples-basic/heap-trace.jsonl exists,
#         is valid JSONL, schema-identical to old mem-profile output

# Diff stub exists and exits cleanly with informative message
cargo run -p lp-cli -- profile diff /tmp/a /tmp/b

# fw-esp32 still builds with renamed feature
cargo check -p fw-esp32 --target riscv32imac-unknown-none-elf \
  --profile release-esp32 --features esp32c6,server

# All workspace tests pass
cargo test --workspace
```

Manual diff comparison: capture an old `mem-profile` output (from
before the change, stored as a baseline fixture) and a new `lp-cli
profile --collect alloc` output of the same workload. `heap-trace.jsonl`
should be byte-identical apart from `meta.json`'s `timestamp` field.

- Files touched: ~15-20.
- No algorithmic complexity. Pure restructuring + new command surface.
