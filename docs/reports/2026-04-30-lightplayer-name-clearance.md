# LightPlayer Name Clearance Report

**Date:** 2026-04-30  
**Researcher:** Agent  
**Scope:** Public web usage, app-store visibility, domain availability, and preliminary trademark risk for releasing an open-source lighting-control app under the name `LightPlayer` in 2026.

## Executive Summary

`LightPlayer` is still practical to release under in 2026, especially as an open-source community project comparable in spirit to WLED. It is no longer a clean, uncontested name, but the conflicts found are not so large or close that they obviously force a rename.

The highest-signal conflicts are not large incumbents. They are small-to-medium software/app uses in adjacent media categories:

- `lightplayer.org` hosts a 2026 privacy policy and terms for a `LightPlayer` media playback / FireTV app.
- `light-player.com` is an active Chinese `Light Player` music-player product with iOS and Android distribution.
- Older Windows download sites list a `LIGHTplayer` / `LIGHTplayer 1.5` audio player from 2008-2009.
- The exact `lightplayer.com` domain is registered and parked via Afternic, not obviously in active product use.

The trademark picture is moderately favorable on a preliminary search:

- A U.S. `LIGHTPLAYER` registration existed for footwear, but it is dead/cancelled and unrelated to software.
- A U.S. `DIRECT LIGHT PLAYER` application existed for software controlling color and intensity of lighting fixtures in connection with electronic games. It is dead/abandoned. This is highly relevant historically, but not a live federal registration.
- I did not find a live exact `LIGHTPLAYER` U.S. software trademark in the sources checked.

The practical recommendation is: keep using `LightPlayer`, pair it with a clear lighting/open-source descriptor, and preserve evidence of first use. Since the project is currently a gift to the community rather than a heavily funded commercial launch, the practical risk is lower than it would be for a closed commercial app. The name should still be treated as **medium trademark/search risk** if later monetization becomes important, mainly because exact and near-exact app names already exist in media playback and because the words `light` + `player` are somewhat descriptive for lighting playback/control software.

## Bottom Line

For a lighting-control app, `LightPlayer` has a plausible lane:

- Your use is about lighting control, fixtures, shaders, embedded devices, and visual effects.
- The most visible current conflicts are music/video/media players, not professional lighting control systems.
- No large, obviously dominant `LightPlayer` brand appeared in this research.
- The old lighting-related U.S. application is dead.

The main weakness is market clarity. Users searching for `LightPlayer` will encounter music/media apps, and app stores may surface the Chinese `Light Player` product or the FireTV/media `LightPlayer` policy before yours. The name can work, but it will need deliberate positioning such as:

- `LightPlayer Lighting Control`
- `LightPlayer Studio`
- `LightPlayer FX`
- `LightPlayer for LEDs`
- `LightPlayer Shader Lighting`

If the intent later shifts toward meaningful monetization, file early in the relevant classes and consider whether the community project should remain `LightPlayer` while any paid offering uses a more distinctive sub-brand.

## Product Context

The intended `LightPlayer` release is an open-source lighting-control application and embedded shader/JIT system, offered primarily as a community tool. Monetization is possible later, but not the current center of gravity.

That context matters:

- Open-source community projects can often tolerate more naming noise than consumer paid apps because users arrive through GitHub, documentation, hardware communities, Discord/forums, and project references rather than app-store keyword search alone.
- The comparison point is closer to WLED than to a proprietary media-player app: the brand value is in community adoption, device support, docs, examples, and trust.
- Trademark hygiene still matters because later monetization, sponsorships, hosted services, paid mobile apps, hardware bundles, or cloud features can raise the stakes.

This makes `LightPlayer` a more reasonable name than it would be for a new paid app launching cold into app stores, but it still argues for clear positioning: `open-source lighting control`, `LED visual effects`, `shader-based lighting`, and `embedded controller` should appear near the name wherever possible.

## Domain Findings

Exact-match domain availability was checked through a mix of RDAP, WHOIS, DNS nameserver lookups, and live page fetches. Treat availability as a point-in-time technical signal, not a purchase guarantee.

**Registered / in active use**

- `lightplayer.com`: registered since 2009-12-15, expires 2026-12-15, nameservers `ns1.afternic.com` / `ns2.afternic.com`. This appears parked/for sale rather than a product site.
- `lightplayer.org`: registered 2025-09-14 through Cloudflare, expires 2026-09-14, active Cloudflare nameservers, hosts the media/FireTV app policy.
- `lightplayer.app`: registered by the project after the initial availability check. This is now the strongest exact-match domain for the community app.
- `light-player.com`: active website for the Chinese `Light Player` music app.

**Appeared unregistered or without DNS nameservers in checks**

- `lightplayer.net`: WHOIS returned no match.
- `lightplayer.dev`: Google Registry RDAP returned 404; no DNS NS records found.
- `lightplayer.io`: WHOIS returned domain not found; no DNS NS records found.
- `lightplayer.co`: WHOIS returned domain not found; no DNS NS records found.
- `lightplayer.ai`: WHOIS returned domain not found; no DNS NS records found.
- `getlightplayer.com`: WHOIS returned no match.
- `lightplayer.lighting`: no DNS NS records found; RDAP endpoint returned access errors, so registrar confirmation is needed.
- `lightplayer.studio`: no DNS NS records found; RDAP endpoint returned access errors, so registrar confirmation is needed.

Recommended acquisition priority:

1. Use `lightplayer.app` as the primary public domain, since it is exact-match, product-appropriate, and now under project control.
2. Buy `lightplayer.dev` and `getlightplayer.com` if still available.
3. Consider `lightplayer.lighting` or `lightplayer.studio` for positioning, but confirm through a registrar.
4. Monitor or inquire about `lightplayer.com` only if budget allows; Afternic parking often means it may be purchasable but potentially expensive.
5. Avoid relying on `lightplayer.org`, since it is already used by a different app.

## Trademark Findings

This is a preliminary research report, not a legal opinion. A trademark attorney should run a full clearance search before filing or major launch.

### U.S. Exact / Near-Exact Marks

**`LIGHTPLAYER`**

- Source: <https://trademarks.justia.com/763/56/lightplayer-76356595.html>
- Serial: `76356595`
- Registration: `2705408`
- Status: `710 - Cancelled - Section 8`
- Filing date: 2002-01-08
- Registration date: 2003-04-08
- Goods: footwear, International Class 25
- First use shown: September 2002

Assessment: low conflict. The mark is dead and unrelated to software/lighting. It does show that the exact word has been registrable as a combined term, but it does not clear the current software field.

**`DIRECT LIGHT PLAYER`**

- Source: `tmsearchapi.com` search result for `DIRECT LIGHT PLAYER`; USPTO status-code references checked separately.
- Serial: `78004968`
- Registration: none shown
- Status code: `606`, which USPTO status-code references define as dead/abandoned, no statement of use filed.
- Owner: Color Kinetics Incorporated.
- Goods: software used to control the color and intensity of lighting fixtures in conjunction with electronic games, International Class 9.

Assessment: historically relevant but not a live federal block. This is the most important trademark fact in the report because it is directly in lighting-control software. The abandoned status is favorable, but the record suggests `light player` phrasing has been used descriptively in lighting playback/control contexts.

### International / Aggregator Search Signals

Searches for exact `LightPlayer` and spaced `Light Player` across web-indexed trademark aggregators did not reveal a live, dominant exact mark in software. EUIPO/WIPO/TMview web-search results were noisy and did not surface an exact active `LIGHTPLAYER` software mark in the checked snippets.

This absence should not be treated as clearance. It means only that a quick public search did not surface a clear blocker.

### Distinctiveness

`LightPlayer` is a reasonably intuitive name for this product, but that is also its weakness:

- `Light` directly describes the lighting domain.
- `Player` can describe playback of shows, effects, timelines, scenes, media, or generated output.
- In lighting control, a "player" can plausibly mean a show playback device or runtime.

The combined word may be registrable, but likely with narrower protection than a coined mark. Expect more coexistence with other `Light ___` and `___ Player` marks than with a highly distinctive name.

## Practical Release Risk

### Main Risks

**Search/app-store confusion**

Users searching `LightPlayer` may find unrelated media players. This matters most if the product ships as a mobile app, especially in global app stores.

**Domain weakness**

The exact `.com` is not controlled, and `.org` is actively used by a different app. Registering `lightplayer.app` materially improves the situation, but users may still assume `.com` or `.org` if they hear the name without a link.

**Trademark narrowness**

The mark is somewhat descriptive in lighting control. Even if registrable, it may not give broad enforcement rights.

**Adjacent software overlap**

The exact current conflicts are media software, not lighting, but all are still software. App stores and trademark examiners often care about the goods/services descriptions, so careful drafting matters.

### Mitigating Factors

- The project has apparent historical continuity back to 2014, which may help priority/common-law narrative if documented.
- The project is open-source and community-oriented, which lowers immediate commercial conflict pressure and gives users more context than a bare app-store listing.
- The project now controls `lightplayer.app`, a strong exact-match domain for an application.
- The current repo already has public search presence for lighting-control/JIT shader use.
- The most relevant U.S. lighting-control trademark record found is dead/abandoned.
- No major lighting-console, DMX, Art-Net, LED pixel-control, or stage-lighting incumbent surfaced using the exact `LightPlayer` name.

## Recommendation

Proceed with `LightPlayer` for the open-source community project, while treating it as a medium-risk, medium-distinctiveness brand if later monetization becomes material.

Recommended next steps:

1. Make `lightplayer.app` the canonical home and link it consistently from GitHub, docs, package metadata, examples, and social profiles.
2. Standardize the public name as `LightPlayer` plus a strong descriptor, for example `LightPlayer Open-Source Lighting Control`, `LightPlayer for LEDs`, or `LightPlayer Shader Lighting`.
3. Preserve evidence of first use: 2014 screenshots, commits, release artifacts, domain records, videos, and public posts.
4. Buy `lightplayer.dev` and `getlightplayer.com` if still available, plus `lightplayer.studio` / `lightplayer.lighting` if they fit the branding.
5. File U.S. trademark applications before major monetization, hardware packaging, paid app-store launch, hosted services, or sponsorship-heavy branding. Likely classes are downloadable software / recorded software in Class 9 and SaaS or hosted control services in Class 42 if applicable. Consider Class 11 only if branding is used directly on lighting hardware/fixtures, and Class 41 only if live performance/show services become part of the offering.
6. Avoid presenting the product as a media player. Lead every public page with lighting-control terms: `open-source`, `lighting`, `LED`, `DMX/Art-Net` if applicable, `shader effects`, `fixtures`, `visual effects`, `embedded controller`.

If maximum trademark strength becomes the priority, a more distinctive paid/commercial sub-brand would be safer. For the community project itself, continuity and product fit make `LightPlayer` a reasonable name.
